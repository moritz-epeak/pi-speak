# Port Negotiation — Implementation Brief

You are working in the `pi-speak` repo on the `stream` branch.

## Goal

Implement robust port negotiation for the TTS daemon so it never silently fails when ports are in use.

## Current Behavior

`ensureDaemonRunning()` in `extensions/index.ts` tries ports 7125-7130 in sequence. If all are taken, the daemon fails to start and `daemonReady` stays false. The user sees no clear error.

## Desired Behavior

1. **Try fallback ports 7125-7130** — same as now, but log which ports were tried
2. **If all taken, try random high ports 7200-7999** — pick up to 5 random ports in this range
3. **If all fail, log a clear error** — "All ports in use. No available port in 7125-7130 or 7200-7999."
4. **Persist the bound port** — once a port is found, always try that port first on restart
5. **Config override** — allow `port` in `~/.pi/agent/speak.json` to force a specific port

## Implementation Details

- Modify `ensureDaemonRunning()` in `extensions/index.ts`
- Add `FALLBACK_PORTS` array (already exists: `[7125, 7126, 7127, 7128, 7129, 7130]`)
- Add `HIGH_PORTS` range: 7200-7999
- Add `persistedPort` variable that remembers the last successful port
- Try `persistedPort` first, then fallback ports, then random high ports
- Update config loading to read `port` override
- Update README.md with port negotiation behavior
- Add tests for port fallback logic
